Introduction

CSS3 Shapes & HSLA Colors

CSS`3 (three) provides new tools that allow for unique shapes and patterns. You can create shapes for the background or as an object in some other area of the page. HSL was introduced in CSS3 and the hsla() function as well. HSL color system was designed in the 1970s (by computer graphics researchers) to align with the way human vision perceives color making attributes. The HSL model was designed to resemble more perceptual color models such as the Natural Color System (NCS).

CSS Shapes

Starting with CSS, lets consider shapes formed with css, as an observation; squares, rectangles, rounded rectangles, ovals, circles and triangles are the general shapes seen on many webpages. To make a shape in CSS, we will use a div and give it an ''ID'' name as follows: <div id="name"></div>

Square shape

HTML code: <div id="square"></div>

Starting with (↑) the div for our square. The Square shape is often used as a background or content boxes frequently. To form the square, define equal height and width for the div element we want to style and set a background color.

  • CSS code:
    #square {
    width: 100px;
    height: 100px;
    background: hsla(30, 50%, 40%, 1);
    }
    Square

Rectangle shape

3 (three) common rectangles,

  • html code: <div id="rectangle1"></div>

    1(one (↑)). This is the typical rectangle. To form the rectangle, define unequal (generally twice the width as the height) height and width for the div element we want to style and set a background color.

    CSS code:
    #rectangle1 {
    width: 200px;
    height: 100px;
    background: hsla(30, 50%, 40%, 1);
    }
    Rectangle1
  • html code: <div id="rectangle2"></div>

    2(two (↑)). This is a rectangle with rounded corners. To form the rectangle, define unequal (generally twice the width as the height) height and width for the div element we want to style, then set the border-radious using px, em, %, rem (e.g. 10px, .25em, 10%, or 0.1rem, regardless of which value type you decide on, each will have an impact on how the corners look) and set a background color.

    CSS code:
    #rectangle2 {
    width: 200px;
    height: 100px;
    background: hsla(30, 50%, 40%, 1);
    border-radius: 10px;
    }
    Rectangle2
  • html code: <div id="rectangle3"></div>

    3(three (↑)). This is a rectangle with tapered corners. To form the rectangle, define unequal (generally twice the width as the height) height and width for the div element we want to style, then set the border-radious using %, (e.g. 25%, this will make the top and bottom taper longer than the sides) and set a background color.

    CSS code:
    #rectangle3 {
    width: 200px;
    height: 100px;
    background: hsla(30, 50%, 40%, 1);
    border-radius: 25%;
    }
    Rectangle3

Circle shape

HTML code: <div id="circle"></div>

Continuing with (↑) the div for our circle. The circle is like the square shape, a circle also needs to have equal width and height. Set the border-radius to 50%. If you want to use pixel for the border-radius value then it is half the height or width value for the div element we want to style (e.g. border-radius: 60px, for this circle) and set a background color.

  • CSS code:
    #circle {
    width: 120px;
    height: 120px;
    background: hsla(30, 50%, 40%, 1);
    border-radius: 50%;
    }
    Circle

Oval shape

HTML code: <div id="oval"></div>

Setting up the (↑) div for an oval. Oval is similar to a circle shape. The oval is rectangular in shape and will need a radius that is half of the height. Set the border-radius to 50%. If you want to use pixel for the border-radius value then it is half the width / height value for the div element we want to style (e.g. border-radius: 100px / 50px, for this oval) and set a background color.

  • CSS code:
    #oval {
    width: 200px;
    height: 100px;
    background: hsla(30, 50%, 40%, 1);
    border-radius: 100px / 50px;
    }
    Oval

Triangle shape

Starting with triangles, this is not complicated and is easier than it may seem. Just define the width and height of the triangle as 0. Then, manipulate the border property of three sides of the triangle ommitting the side for the point of the triangle. Define a base border value and add border color while defining smaller (half the base) but equal values for the other two sides with "border color" defined as "transparent". Make the triangle point to different sides, simply by changing the border properties.

  • html code: <div id="triangle_up"></div>

    1(one (↑)). Triangle up...

    CSS code:
    #triangle_up {
    width: 0;
    height: 0;
    border-left: 50px solid transparent;
    border-right: 50px solid transparent;
    border-bottom: 100px solid
           hsla(30, 50%, 40%, 1);
    }
    Up
  • html code: <div id="triangle_down"></div>

    2(two (↑)). Triangle down...

    CSS code:
    #triangle_up {
    width: 0;
    height: 0;
    border-left: 50px solid transparent;
    border-right: 50px solid transparent;
    border-top: 100px solid
           hsla(30, 50%, 40%, 1);
    }
    Down
  • html code: <div id="triangle_left"></div>

    3(three (↑)). Triangle left...

    CSS code:
    #triangle_left {
    width: 0;
    height: 0;
    border-top: 50px solid transparent;
    border-bottom: 50px solid transparent;
    border-right: 100px solid
           hsla(30, 50%, 40%, 1);
    }
    Left
  • html code: <div id="triangle_right"></div>

    4(four (↑)). Triangle right...

    CSS code:
    #triangle_right {
    width: 0;
    height: 0;
    border-top: 50px solid transparent;
    border-bottom: 50px solid transparent;
    border-left: 100px solid
           hsla(30, 50%, 40%, 1);
    }

Other Shapes

3 (three) more shapes that are not as common as the other shapes above, but are frequently used as well, Diamond, Trapezium, Parallelogram. These shapes have some additional code necessary to acheive the desired result.

  • html code: <div id="diamond"></div>

    1(one (↑)). Creating a diamond can be displayed several ways. Using the transform property and the rotate value will display two triangles next to each other and set a background color.

    CSS code:
    #diamond {
    width: 100px;
    height: 100px;
    background: hsla(30, 50%, 40%, 1);
    /* Rotate */
    transform: rotate(-45deg);
    /* Rotate Origin */
    transform-origin: 0 100%;
    margin: 8.5% 10% 1.5% 10%;
    }
    Diamond
  • html code: <div id="trapezium"></div>

    2(two (↑)). A trapezium can be displayed by setting up equal left and right borders with a wide bottom (typically double left or right border) and set bottom border solid and set the color. As with a triangle, manipulate the border property of the trapezium ommitting one side.

    CSS code:
    #trapezium {
    width: 100px;
    height: 0;
    border-left: 50px solid transparent;
    border-right: 50px solid transparent;
    border-bottom: 100px solid
           hsla(30, 50%, 40%, 1);
    }
    Trapezium
  • html code: <div id="parallelogram"></div>

    3(three (↑)). The parallelogram is rectangular in shape. To create a parallelogram shape, define unequal (generally twice the width as the height) height and width for the div element we want to style and set a background color. Set up a transform value of skew to turn the element in a 30 degree angle.

    CSS code:
    #parallelogram {
    width: 200px;
    height: 100px;
    transform: skew(30deg);
    background: hsl(30, 50%, 40%, 1);
    }
    Parallelogram

HSLA Colors

Having discussed multiple shapes, lets now look at the HSLA color system as it applies to web design and the hsl() and hsla() functions in css. HSLA stands for hue, saturation, lightness, and alpha.
  Hue, the attribute of a visual sensation according to which an area appears to be similar to one of the perceived colors: red, yellow, green, and blue, or to a combination of two of them, consider a color wheel or slider. At 0° and 360° are reds. 120° are greens and 240° are blues. Use anything in between 0-360. Values above and below will be modulus 360.
  Saturation, The radiant power of light passing through a particular surface area, starting at 0% is no color, completely desaturated (grayscale) and 100% the whole color, fully saturated (full color).
  Lightness, the radiance weighted by the effect of each wavelength on a typical human observer, starting at 0% is completely dark (black) and 100% is completely light (white). 50% is normal lightness.
  Alpha, sets the Opacity/Transparency value. 0 is fully transparent. 1 is fully opaque. 0.5 is 50% transparent.

Similar to rgba in HSLA you declare three values determining the color and then a fourth value for its transparency level. The following css code fragment(s) shows the use of HSLA format :

h1{ background-color: hsla(30, 50%, 40%, 0.0);}
h2{ background-color: hsla(30, 50%, 40%, 0.2);}
h3{ background-color: hsla(30, 50%, 40%, 0.4);}
h4{ background-color: hsla(30, 50%, 40%, 0.6);}
h5{ background-color: hsla(30, 50%, 40%, 0.8);}
h6{ background-color: hsla(30, 50%, 40%, 1.0);}

  In the preceding code fragment, the opacity or alpha value 1.0 or 1, shows the color of the H6 element 100%, while the opacity value 0.0 of the H1 element hides the color of the element. If your not well versed with color, hsla() allows for some simple rules to create nice effects.

.this-box {
color: hsla(60, 20%, 100%, 1);
background-image: linear-gradient(to bottom right,
hsla(60, 20%,  0%, 1), hsla(60, 20%, 10%, 1),
hsla(60, 20%, 20%, 1), hsla(60, 20%, 30%, 1),
hsla(60, 20%, 40%, 1), hsla(60, 20%, 50%, 1),
hsla(60, 20%, 60%, 1), hsla(60, 20%, 70%, 1),
hsla(60, 20%, 80%, 1), hsla(60, 20%, 90%, 1),
  );
}

Looking at the real appeal of HSLA from the above two examples, is it makes more intuitive sense as to what changing the values will do to the color. Increasing the second value will increase the saturation of that color. Decreasing the third value will decrease the lightness of that color. That makes creating your own color variations much easier. The HSLA system also makes changing color values programatically much easier.

Anothing point to consider is, HSL values work with similar semantics and ranges as rgb, but rather than working with values as the monitor interprets the colors, hsl values work with hue, saturation, lightness values. This looks syntactically similar to rgb values but the ranges are different. This system is based on a Munsell color system (Munsell was the first to separate out color into these three channels, or create a three dimensional system based on mathematical principles tied to actual human vision). For humans, the use of hsl can be a lot easier to interpret.

 - One last point on color -
► Looking at High definition; The number of lines in the vertical display resolution. High-definition television (HDTV) resolution is 1,080 or 720 lines. In contrast, regular digital television (DTV) is 480 lines (upon which NTSC is based, 480 visible scanlines out of 525) or 576 lines (upon which PAL/SECAM are based, 576 visible scanlines out of 625).
► Now consider this one small point. RGB, HEX, and Named colors are all based on the technology of the CRT monitors. HSL was designed from the start to be aligned with human eye sight and how humans view colors.
► Also, there are less common but still popular UltraWide resolutions, such as 2560×1080p (1080p UltraWide). So when it comes to color...When you want your colors being displayed in High Definition...Think HSL = HD.

→ Authors coding tip: Always use HSLA (set the alpha channel to , 1 [unless you need to change the value]) over simply using HSL(), this way you know your color is not transparent and should not be displayed as such. If something does go transparent that was not intended to do so, knowing you used hsla helps to quickly eliminate that as the problem.
